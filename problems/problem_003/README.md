## 프로그램 설명

- 이동 방향을 입력받아, 현재 위치가 출발지로부터 얼마나 떨어져있는지를 계산하는  
프로그램

## 배운 점

- 딕셔너리의 value에 튜플도 저장할 수 있다  
딕셔너리에 튜플을 넣는 이유 :  
1. 좌표, 위치, 방향 같은 복합값을 하나로 묶고 싶을 때
2. 불변의 자료형이 필요할 때 (튜플은 수정 불가)
3. 여러 값을 한꺼번에 반환하거나 저장하고 나중에 개별 요소에  
접근하고 싶을 때 예를 들어, 게임에서 캐릭터 움직임, 지도 좌표, 벡터 방향 등을 저장할 때,  
튜플과 딕셔너리 조합이 많이 쓰인다
4. 예제

**방향과 이동량을 딕셔너리로 정의**
`directions = {`
    `"N": (0, 1),`
    `"S": (0, -1),`
    `"E": (1, 0),`
    `"W": (-1, 0)`
`}`

**캐릭터 초기 위치**
`x, y = 0, 0`
`print(f"시작 위치: ({x}, {y})")`

**이동 명령 시퀀스**
`moves = ["N", "N", "E", "S", "E", "E"]`

**이동 시뮬레이션**
`for move in moves:`
    `dx, dy = directions[move]`  # 딕셔너리에서 좌표 변화 가져오기
    `x += dx`
    `y += dy`
    `print(f"{move} 이동 -> 위치: ({x}, {y})")`

`print(f"최종 위치: ({x}, {y})")`

- sum()대신 정수 누적, 조건문 단순화 등 코드의 가독성 향상 팁


## 새로 알게된 문법

**`continue`**
- 반복문(for, while)에서 사용하는 제어문
- 의미 : 현재 반복을 즉시 끝내고, 다음 반복으로 넘어가라는 뜻
- break와의 차이 : break - 반복문 자체를 완전히 종료  
continue - 이번 반복만 스킵하고 다음 반복으로 이동
- 예제 :  
`for i in range(5):`
    `if i == 2:`
        `continue`
    `print(i)`
출력값 : 0 1 3 4

**`all`**
- all(반복 가능한 것)
- 전부 True면 True, 하나라도 False면 False

**generator expression**
- 제너레이터는 한 번에 모든 값을 메모리에 올리지 않고, 필요할 때 하나씩 생성하는 반복 가능한 객체
- 일반 리스트와 달리, lazy evaluation(필요할 때 계산) 방식으로 동작
- 표현식, 함수 두 가지 방식으로 만들 수 있다.
