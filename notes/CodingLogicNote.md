# 프로그래밍을 할 때 생각해야할 로직

## 개발자가 가져애할 논리 구조
- 생각 먼저 하고 코드를 짤 것
- 큰 문제를 작은 문제로 쪼갤 것  
예 : 문제 : 문자열에서 가장 많이 나온 문자
1. 문자 하니씩 보기
2. 개수 세기
3. 가장 큰 값 찾기
- 상태 저장하기
- 코드 짤 때 순서  (거의 이렇다)
1. 입력 받기
2. 변수 만들기
3. 반복문 돌리기
4. 조건 처리
5. 결과 출력

## if문
- if문 안에 elif를 n번 만들어 n개 모두 작동시키고 싶을 때, 상단의 elif문이 참이 되어  
실행 되는 경우 뒤에 있는 elif문이 작동되지 않으므로 이런 경우 if문을 여러개 만들 것  
- if문의 조건에 따라 실행흐름이 꼬일 수 있으니 조건을 알고리즘에 맞게 정확히 작성할 것

## while문
- 무한반복문 (while True)의 조건을 추가하고 싶을 때 if문으로 무한반복문을 감쌀 필요없이,  
무한반복문 안에 if문을 추가하여 조건을 설정할 수 있다.
